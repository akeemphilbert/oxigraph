# Contributing to Oxigraph

Documentation for people who want to change Oxigraph itself — fix a bug, add a
feature, or improve one of its components.

- **[Tutorials](tutorials/README.md)** — learn by doing: build Oxigraph from source,
  run the tests, and make your first change.
- **[How-to guides](how-to/README.md)** — task recipes: run the conformance
  testsuite, benchmarks, and fuzzers, and work on the Python, JavaScript, and CLI
  components.
- **[Reference](reference/README.md)** — look-up material: the crate map, feature
  flags, and development commands.
- **[Explanation](explanation/README.md)** — how Oxigraph is put together:
  architecture and design.

If you just want to use Oxigraph, the [user documentation](../users/README.md) is
for you.
